supervisor web页面设置

发表于 2017-03-06 | 阅读次数: | 分类于 supervisor |

   由于服务器上部署了太多了服务，然后就导致supervisor服务很多，每次想看一下状态或重启服务的时候，都要跑到服务器上面去操作，略感麻烦。
   然后去google了一下，supervisor官方是有提供ui页面的。

具体操作

配置文件

   默认的配置文件是放在/etc/supervisor/supervisord.conf里面的，当然你也可以用echo_supervisord_conf查看你的supervisord的配置。
然后也可以把结果导入到文件中echo_supervisord_conf > /etc/supervisord.conf。不过一般配置文件的话，也不要修改什么。贴一下我的配置文件。

主要有几个比较重要的地方：

  1. inet_http_server是配置web ui的内容。端口、用户名、密码。
  2. supervisord里面的childlogdir，是用来修改log日志保存的地方，修改这个很有必要，如果是放在容器里面的话，日志肯定是要导出来的。
  3. include的话，就是单独的每个服务配置文件。

nginx

   因为服务器上跑了很多服务，然后也会配了很多的nginx服务，然后在supervisor的nginx服务时，需要注意几个地方，先贴自己的nginx代码。
